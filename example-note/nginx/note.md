# Nginx

## 安装与启动

```shell
sudo apt-get install nginx
sudo nginx
```

## 目录

位于/etc/nginx下

通过查看nginx.conf可以看到引用了其下/sites-enabled中的内容。

所以最终修改的配置文件应该是/etc/nginx/sites-enabled/default

## 项目路径配置


## SSL配置

将.crt和.key文件放置到/home/zekdot/ssl中

```json
server {
    listen 80 default_server;
    listen [::]:80 default_server;
    # SSL configuration
    #
    listen 443 ssl default_server;
    listen [::]:443 ssl default_server;

    ssl on;
    ssl_certificate /home/zekdot/ssl/1_www.zekdot.com_bundle.crt;
    ssl_certificate_key /home/zekdot/ssl/2_www.zekdot.com.key;

```

重启nginx后即可。

## 配置http自动跳转https

```
server {
listen 80;
server_name www.域名.com;
rewrite ^(.*) https://$server_name$1 permanent;
}
server {
listen 443;
server_name www.域名.com;
root /home/www;
ssl on;
ssl_certificate /etc/nginx/certs/server.crt;
ssl_certificate_key /etc/nginx/certs/server.key;
}
```

## 配置Session可用

```json
server {
	listen 9009;
	server_name localhost;

    #添加头部信息
    proxy_set_header Cookie $http_cookie;
    proxy_set_header X-Forwarded-Host $host;
    proxy_set_header X-Forwarded-Server $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
	
    #添加拦截路径和代理地址
    location /api/ {              
        proxy_pass http://localhost:9010/;  #注意：使用代理地址时末尾记得加上斜杠"/"。      
    }
    
	root /home/zekdot/password;

	location / {
		# First attempt to serve request as file, then
		# as directory, then fall back to displaying a 404.
        index index.html;
		try_files $uri $uri/ =404;
	}
}
```
